Give-N trial logic

When reporting trial logic or knower level assignment issues, please include the following:

  1. Version (Titrated or Non-Titrated)
  2. Number of items available; maximum number requested and starting number (if Titrated)
  3. A screenshot of trial history (generated at the bottom of test page)

Knower level assignment

Knower level assignment for both the Titrated and Non-Titrated versions of Give-N are based on Wynn (1992).

Subset knowers

  1. If child correctly gives N at least 2 out of 3 times when requested (where N is not the maximum number tested), AND
  2. If child does not give that N incorrectly for another requested number more than once,
  3. Then child is identified as N-knower.

CP knowers

  1. If child correctly gives N at least 2 out of 3 times when requested (where N is the maximum number tested)
  2. Then child is identified as CP knower.

Titrated Give-N Trial Logic

Trial order is determined via a staircase function: for a given trial, if a child correctly gives N, the next number requested is N+1. If a child incorrectly gives N, the next number requested is N-1. Note that this staircasing procedure may be broken if a child demonstrates that they do not understand either N or N-1 (e.g., if child correctly gives 3 when asked for 3, but also gives 3 for another number more than once, the program will loop back to 2).

Non-Titrated Give-N Trial Logic

Each number (specified by user in test setup) is tested three times, with a randomized order within each set (e.g., if the numbers to be tested are 1-5, program shuffles those numbers for first test, then shuffles again for second and third tests). Each number is tested three times, regardless of performance.
